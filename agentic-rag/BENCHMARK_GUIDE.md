# Multi-Tenant Agentic RAG vs Conversational RAG: Benchmark Metrics

## Overview
This benchmark compares two RAG architectures for HR assistance:
1. **Conversational RAG** (Baseline): Traditional retrieval-only system
2. **Multi-Tenant Agentic RAG** (Proposed): Tool-augmented multi-agent system

## Key Metrics Being Measured

### 1. Performance Metrics
- **Success Rate**: Percentage of queries answered successfully
- **Answer Accuracy**: How many answers contain expected information
- **Average Latency**: Mean response time in seconds
- **Median Latency**: Median response time for robustness
- **Answer Length**: Average characters in responses (indicator of detail level)

### 2. Capability Metrics by Query Complexity

#### Level 1: Simple Retrieval
- Queries: "How many casual leave days?", "What are working hours?"
- Tests: Basic information extraction from documents

#### Level 2: Multi-Step Reasoning
- Queries: "Can I carry forward leave?", "What's the approval process for 5+ days?"
- Tests: Understanding multiple policy sections and reasoning

#### Level 3: Action Required (Agentic-Only)
- Queries: "Check my leave balance", "What holidays are coming up?"
- Tests: Tool integration and real-time data access

#### Level 4: Ambiguous Queries
- Queries: "I'm not feeling well", "Tell me about leaves"
- Tests: Intent understanding and helpful clarification

### 3. System Capabilities

| Capability | Conversational RAG | Agentic RAG |
|-----------|-------------------|-------------|
| Information Retrieval | ✓ | ✓ |
| Multi-Step Reasoning | ✓ | ✓ |
| Action Execution | ✗ | ✓ |
| Tool Integration | ✗ | ✓ (11 tools) |
| Multi-Company Support | ✗ | ✓ |

## Expected Improvements

### Hypothesis
Agentic RAG will demonstrate:
1. **Higher accuracy** on complex queries (20-40% improvement)
2. **Broader capability** (100% improvement on action queries: 0% → 100%)
3. **Better ambiguity handling** due to tool-based clarification
4. **Slightly higher latency** (acceptable trade-off for capabilities)

### Conference Paper Angle
- **Problem**: Static RAG systems can't execute actions or access real-time data
- **Solution**: Multi-agent architecture with tool integration
- **Innovation**: Multi-tenant isolation for enterprise deployment
- **Results**: Demonstrate quantified improvements across metrics
- **Use Case**: HR onboarding assistant (practical, relatable, high-impact)
